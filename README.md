# Character Sheets — Shadowdark & Cyberpunk RED

Лёгкое приложение с синхронизируемыми листами персонажей для пати. Любой с ID
кампании видит чужие листы, но редактирует только свои (по паролю). У кампании
есть общий журнал (квесты/заметки), который правит только ГМ.

- **Бэкенд:** FastAPI + файловое хранилище JSON (`data/`). Один воркер.
- **Фронтенд:** чистый статик (vanilla JS, без сборки).
  - `/` — Shadowdark (тема «пергамент/блэклеттер»).
  - `/cyberpunk.html` — Cyberpunk RED (тема «DataTerm», ч/б терминал).

## Локальный запуск (всё в одном процессе)

```bash
python -m venv venv
venv/Scripts/python -m pip install -r requirements.txt   # Linux/Mac: venv/bin/python
python run.py                                            # http://127.0.0.1:8000
```

FastAPI отдаёт и API, и статику — открывай `http://127.0.0.1:8000`.

## Раздельный хостинг (фронт на GitHub Pages, бэк на сервере)

Фронт и бэк независимы — фронт ходит на бэк по `API_BASE`.

**1. Бэкенд на своём сервере:**
```bash
# прод-режим, без --reload, слушаем наружу
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000
# ограничить CORS своим origin Pages (через запятую можно несколько):
ALLOWED_ORIGINS="https://USER.github.io" python -m uvicorn app.main:app --host 0.0.0.0 --port 8000
```
Поставь его за reverse-proxy с HTTPS (nginx/Caddy) — браузер не пустит
`https`-страницу на `http`-API.

**2. Фронтенд на Pages:** залей содержимое папки `static/` в репозиторий Pages и
в `static/config.js` укажи адрес бэка:
```js
window.SD_CONFIG = { apiBase: 'https://api.example.com' };  // без слэша на конце
```

Готово: `EventSource` и все запросы пойдут на `apiBase`, синхронизация работает
кросс-доменно.

## Системы

Кампания хранит поле `system` (`shadowdark` | `cyberpunk_red`). Страница CP:RED
создаёт кампании со своей системой; если открыть «чужую» кампанию не на той
странице — покажется подсказка со ссылкой на нужную.

## Замечания

- Локи и SSE живут в памяти процесса → запускать **одним воркером**.
- Авторизация — хеш пароля в теле запроса (не куки); секреты на сервере хранятся
  двойным хешем и наружу не отдаются.
- Броски кубов — полностью на клиенте, на сервер не уходят.
